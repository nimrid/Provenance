# Provenance: Zero-Knowledge Luxury Goods Authentication

Provenance utilizes zero-knowledge proofs on the Stellar network to establish unbreakable, privacy-preserving chains of ownership for luxury goods. 

With Provenance, a luxury good’s physical identity (its hardware serial number) and the owner’s identity are completely decoupled from the public blockchain. Your identity remains hidden. The authenticity remains absolute.

---

## 🌟 Key Features

1. **Zero-Knowledge Provenance (Noir & Barretenberg)**:
   - Physical items are tied to the blockchain using cryptographic commitments (hashes of a physical serial number and a secret nonce). 
   - Operations (transfers, reporting stolen) are verified on-chain via an UltraHonk verifier using proofs generated by Barretenberg.
   - **Privacy First**: The serial number and the user's secret are never published on the blockchain.

2. **Secure On-Chain Settlement (Soroban on Stellar)**:
   - All state transitions (genesis minting, ownership transfers, and stolen flags) are governed by a Soroban smart contract.
   - Fast, low-fee settlement using Stellar.

3. **"Report Stolen" ZK Flow**:
   - If a physical item is stolen, the true owner can use their secret nonce to generate a ZK proof flagging the item as stolen.
   - This prevents any future transfers of the item and permanently marks the commitment as compromised, without revealing the underlying serial number.

4. **NFC Tag Integration**:
   - WebNFC integration (supported on Android Chrome) allows users to seamlessly scan embedded NFC chips in physical luxury goods to autofill serial numbers.

5. **Responsive Luxury Interface**:
   - Built with Next.js, featuring a dark-themed, mobile-responsive luxury aesthetic.

## 🏗️ Architecture

### 1. Smart Contract (`/contracts/provenance`)
Written in Rust using the Soroban SDK. The contract maintains the public registry of commitments.
- **Verification Keys**: Genesis, Transfer, and Stolen UltraHonk Verification Keys are initialized immutably.
- **State mappings**: `Commitment` (valid items), `Nullifier` (spent items), and `Stolen` (flagged items).

### 2. Zero-Knowledge Circuits (`/circuits`)
Written in Noir.
- **`genesis`**: Computes the initial hash commitment.
- **`hash_transfer`**: Proves knowledge of an existing item and generates a new commitment and nullifier.
- **`report_stolen`**: Proves knowledge of an existing item and outputs the commitment to be flagged.

### 3. Web Client (`/client`)
Built with Next.js and React.
- Interacts directly with the Soroban RPC and the Stellar Freighter wallet.
- **Important Hackathon Note on Proving**: For this demo, proof generation runs on a trusted backend server (`/api/prove/*`). This means the user's private inputs briefly transit to the server during proving. We've scoped this as an accepted hackathon trade-off: the ZK circuit, the proof verification, and the on-chain settlement are all cryptographically real. The server is completely stateless and handles temporary cleanup securely via `finally` blocks (secrets are never logged or stored). Moving the prover to client-side WASM so secrets never leave the device is a documented V2 milestone.

## 🚀 Getting Started

### Prerequisites
- [Node.js](https://nodejs.org/en/) (v18+)
- [Rust](https://www.rust-lang.org/tools/install) (latest stable)
- [Stellar CLI](https://developers.stellar.org/docs/build/smart-contracts/getting-started/setup)
- [Nargo & bb (Barretenberg)](https://noir-lang.org/docs/getting_started/installation)

### 1. Build and Deploy Smart Contract
Start a local Stellar network (Quickstart) or use the Testnet.

```bash
cd contracts/provenance
# Build the contract to WASM
stellar contract build

# Deploy and initialize the Verification Keys
./deploy.sh
```
*Note: The `deploy.sh` script will automatically compile the Noir circuits, extract the UltraHonk verification keys, deploy the WASM to Soroban, initialize the contract with the keys, and update your client's `.env.local`.*

### 2. Run the Next.js Client
```bash
cd client
npm install
npm run dev
```
Navigate to `http://localhost:3000` to access the application.

## 📖 User Flows

### A. Manufacturer (Minting)
1. Navigate to `/admin`.
2. Scan the NFC tag of a new luxury item or manually enter its Serial Number.
3. Generate the proof. The app will return a randomly generated **Secret Nonce**.
4. The Manufacturer securely delivers this Secret Nonce to the buyer.
5. Register the genesis commitment on the Stellar blockchain.

### B. Owner (Transfer)
1. Navigate to `/transfer`.
2. Input the item's Serial Number, your Old Secret Nonce, and generate a New Secret Nonce.
3. The app generates a ZK proof to safely transfer ownership on-chain, nullifying your old commitment and establishing the new buyer's commitment.

### C. True Owner (Reporting Stolen)
1. Navigate to `/stolen`.
2. Input the Serial Number and your Secret Nonce.
3. The ZK proof allows you to securely flag the item's commitment as stolen on-chain.
4. Any future buyers attempting to verify the commitment will see a massive `REPORTED STOLEN` warning.

### D. Buyer (Verification)
1. Navigate to `/verify`.
2. Enter the public commitment hash provided by the seller.
3. The app queries the Soroban contract and validates if the item is Authentic & Active, Invalid/Spent, or Reported Stolen.
